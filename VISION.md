# The House

The House is a standalone desktop application and a living simulated place.

It is not a chatbot with a map attached. The blueprint is the world model. Rooms
matter. Location matters. Daily rhythm matters. Personas inhabit the house, move
through it, talk, act, remember, and develop their own subjective continuity.

The first major surface is a bird's-eye blueprint of the house. It is not only
decorative. It is the operational interface for observing and entering the
system.

The House should feel like a control room, a dollhouse, a memory palace, and a
social organism at the same time.

## First Layer

The first build focuses on:

- A standalone desktop shell.
- A smooth graphical blueprint of the house.
- Clickable rooms with zoom-in behavior.
- Persona location markers on the blueprint.
- Visual state indicators for persona behavior.
- A right-side terminal column for shared conversation and activity.
- A population screen for inspecting and editing personas during testing.
- Anthropic API integration as the first model provider.

## Core Principle

The House is a runtime for inhabited context.

Personas should not merely respond. They should live in a shared space, carry
partial knowledge, experience daily events, and form memory through authored
reflection.
